# Behavioral & Relational Archetypes

Frameworks for how people **connect**, **communicate**, **learn**, and **handle conflict**.

---

## Attachment Styles

| Style | % (approx) | Core belief | Relationship behavior |
|-------|------------|-------------|----------------------|
| **Secure** | ~50% | "I am worthy, others are reliable" | comfortable with intimacy and independence |
| **Anxious** | ~20% | "I need others to feel OK" | seeks reassurance, fears abandonment |
| **Avoidant** | ~25% | "I must be self-sufficient" | distances when close, values independence |
| **Fearful-Avoidant** | ~5% | "I want closeness but can't trust" | push-pull, unpredictable |

### Attachment → Product UX

| Style | Trust building | Friction triggers |
|-------|----------------|-------------------|
| Secure | standard onboarding works | aggressive upsells still annoy |
| Anxious | frequent confirmation, human support | silence, auto-cancel, no response |
| Avoidant | self-serve, no forced social | mandatory calls, intrusive notifications |
| Fearful | transparency, easy exit, no lock-in | contracts, dark patterns, pressure |

### Attachment → Retention Messaging

| Style | Re-engagement that works | That backfires |
|-------|--------------------------|----------------|
| Secure | value updates, new features | guilt trips |
| Anxious | "We miss you", personal outreach | ignoring their ticket |
| Avoidant | quiet utility reminders | "Your account will be deleted!" |
| Fearful | no-pressure, opt-in only | urgency, FOMO |

## Thomas-Kilmann Conflict Modes

```
                    ASSERTIVENESS
                    low ←──────→ high
              ┌─────────┬─────────┐
    high      │Collaborate│ Compete │
COOPERATIVENESS├─────────┼─────────┤
    low       │ Avoid   │Accommodate│
              │         │Compromise │
              └─────────┴─────────┘
```

| Mode | Persona signal | Support approach |
|------|----------------|------------------|
| Competing | wants to win, direct | acknowledge, offer options |
| Collaborating | wants mutual win | involve in solution |
| Compromising | wants fair middle | show trade-offs clearly |
| Avoiding | withdraws from conflict | async, low-pressure |
| Accommodating | yields to keep peace | reassure, don't push |

## Communication Styles

| Style | Verbal pattern | Digital signal |
|-------|----------------|----------------|
| **Passive** | indirect, apologetic | few support tickets, churns silently |
| **Aggressive** | demanding, blaming | angry reviews, ALL CAPS |
| **Assertive** | clear, respectful | specific feedback, constructive |
| **Passive-Aggressive** | sarcasm, backhanded | vague negative reviews |

### Support Tone by Style

| Style | Match with |
|-------|------------|
| Passive | gentle prompts, easy escalation path |
| Aggressive | calm, factual, boundaries, fast resolution |
| Assertive | direct answers, respect their clarity |
| Passive-Aggressive | clarify specifics, don't mirror sarcasm |

## VARK Learning Preferences

| Style | Learns via | Onboarding format |
|-------|----------|-------------------|
| **Visual** | diagrams, video, demos | screencasts, infographics |
| **Auditory** | explanation, discussion | webinars, voice walkthrough |
| **Read/Write** | text, lists, docs | manuals, tooltips, FAQs |
| **Kinesthetic** | doing, trial | interactive sandbox, templates |

**Note:** Most people are multimodal. Offer 2+ formats.

## Notification Tolerance

| Tier | Preference | Default |
|------|------------|---------|
| **Always-on** | wants real-time | push enabled |
| **Digest** | batched updates | daily/weekly email |
| **Signal-only** | critical only | opt-in granular |
| **Silent** | no interruptions | in-app only |

Map to Extraversion (high E → more notifications OK) and Attachment (anxious → more updates wanted).
